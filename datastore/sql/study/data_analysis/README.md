## 데이터 분석이란

#### 빅데이터 주목 이유
```
기억용량의 대용량화
하드웨어 가격의 하락과 클라우드 기술의 등장
분산처리를 담당하는 미들웨어의 당장
```

#### 용도

```
개별적인 사용자의 행동 파악
사용자의 흥미와 기호를 기반으로 하는 추천
광고 타깃팅
예측 모델링
기계 학습
```

#### 데이터의 종류

```
업무 데이터
서비스와 시스템을 운용하기 위한 목적으로 구축된 데이터베이스에 존재하는 데이터
트랜잭션 데이터와 마스터 데이터로 분류됨

- 트랜젝션 데이터 
	구매, 리뷰, 게임 플레이 처럼 서비스와 시스템을 통해 사용자의 행동을 기록한 데이터

- 마스터 데이터
	카테고리, 상품처럼 서비스와 시스템이 정의하고 있는 데이터
	
로그 데이터
- 통계 또는 분석을 주 용도로 설계된 데이터
- 특정 태그를 포함해서 전송된 데이터
- 특정 행동을 서버 측에 출력한 데이터
중요한 것은 '누적형' 데이터라는 것이다.(기존의 데이터를 수정하지 않음)
```

#### 업무 데이터

```
- 특징
데이터의 정밀도가 높다. (트랜잭션, 롤백 기능으로 데이터의 정합성을 보장)
갱신형 데이터 (CRUD)
다뤄야 하는 테이블의 수가 많다. (정규화, ERD)

- 활용
매출액, 사용자 수처럼 정확한 값을 요구할 경우 활용하기
서비스의 방문 횟수, 페이지뷰, 사용자 유도 등의 데이터 분석에는 사용할 수 없음
데이터 변경이 발생할 수 있으므로 추출 시점에 따라 결과가 변화할 수 있음
```

#### 로그 데이터

```
- 특징
시간, 사용자 엔드포인트, IP, URL, 레퍼러, Cookie 등의 정보 저장하기
로그 데이터는 추출방법에 따라 데이터의 정밀도가 달라짐
계속 기록을 추가하는 것뿐이므로 과거의 데이터가 변경되지는 않음

- 축적방법
태그, SDK를 통해 사용자 장치에서 데이터를 전송하고 출력하기(비컨 형태) : GA처럼 HTML에 특정 태그를 집어놓고 데이터를 전송하는 형식을 나타내는 방법, 크롤러의 영향을 적게 받음
서버에서 데이터를 추출하고 출력하기(서벼 형태) 

- 활용
사이트 방문 횟수, 페이지뷰, 사용자 유도 상황을 집계하고 분석할 때 주로 사용
최신 상태를 고려한 분석에는 적합하지 않음
계속 기록을 누적하는 형태이므로 추출 결과가 변할 가능성이 적음(업무데이터의 경우 레코드를 제거할 가능성)
데이터의 정확도는 업무 데이터에 비해 낮음(크롤러의 로그가 함께 포함되어 집계될 가능성 등)
```

```
로그데이터는 주로 웹사이트에서의 행동을 기록할 때 활용
업무데이터는 웹사이트 뿐만 아니라 오프라인에서의 데이터도 사용 가능

따라서 둘을 활용하면 웹사이트에서의 행동이 오프라인의 행동에 어떠한 영향을 미치는지 등을 조사할 수 있다.
```

#### 데이터 사용가치

```
- 목표를 관리하고, 설계하고, 서비스/조직의 성장에 기여하기 (목표관리)
매출, 접근 수, 사용자 수처럼 서비스가 지향하는 목표가 현재 어느정도 진행되었느지 파악하고 대책 검토

- 사용자 행동을 기반으로 경향을 발견하고, 매출과 서비스 개선에 기여하기(서비스 개선)
- 과거의 경향을 기반으로 미래의 행동 예측하기 (미래 예측)
```
